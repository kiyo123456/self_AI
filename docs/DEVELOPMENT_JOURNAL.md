# 開発日誌

---

## 記録の方針

- 正解を書く場所ではない。試行錯誤のプロセスを書く場所。
- 「最初はAだと思ったけど、Bだった」という差分を記録する。
- AIに指摘された差分を内省し、次のタスクの精度を上げる材料にする。
