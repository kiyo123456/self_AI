## アピールポイントの整理

このプロジェクトが他のポートフォリオと差別化できる点をまとめる。

### 技術力
- RDF・SPARQL を正しく理解・実装している
- DDD の設計原則を守ってアーキテクチャを構築している
- Gemini API と Laravel Queue を使った非同期推論を実装している

### 設計思考
- 「なぜこの技術を選んだか」を説明できる
- 「なぜこの構成にしたか」を説明できる
- トレードオフを意識して判断している

### 拡張性
- 外部LODとの統合を見据えた設計になっている
- LLMを差し替えられる設計になっている
- 推論結果を RDF で保存することで説明可能性を担保している

## 「AIと一緒に作った」ことの説明

> 「これはAIに作ってもらったんですか？」

**準備すること**
- AIを「ペアプログラマー」「拡張機能」として使ったことを明確に説明する
- 「設計の意思決定は自分で行った」ことを具体例とともに示す

#### 回答例

「AIをペアプログラマーとして活用しました。
設計の意思決定——例えば『ExperienceのIDをMySQLで管理する』『スキルは推論で導出する』といった判断——は全て自分で行っています。
AIには主にインフラ層の実装やSPARQLクエリの構文を任せ、ドメイン層は自分の手で書きました。
AIに依存するのではなく、AIを『自分の思考を拡張するツール』として使うことで、設計思考を訓練しながら開発を進めました。」
